# 人力成本筛选枚举（cost_enum）

> **定位**：集中维护 **`bizId=119`** 人力成本取数时，用户常用说法与 **`condition` / `dimensions`** 所需编码、枚举之间的对照。避免在 `SKILL.md` 或指标清单里堆长表；**新增一类映射时在本文件增加同级 `##` 章节即可**（不必为每张表单独建文件，除非单章过长再拆子目录）。  
> **通用规则**：多值归类筛选优先 **`filterType: 10`（INCLUDE）**；维度 alias 以 `measures_catalog.md` 对应模型「分析维度」表为准；线上枚举冲突时以 **`data_search_dimension_values`** 为准并回写本文档。

## 员工类型（`real_emp_cls_cd`）

> **用途**：用户按「正式 / 实习生 / 外包 / 其他」等口语查人、筛明细时，展开为维度 **`real_emp_cls_cd`** 的取值集合。描述列 **`emp_cls_cd_desc`** 见指标清单；语义不稳时优先用编码展开。

**维护说明**：编码集合以人事 / 主数据口径为准。

### 归类与编码对照

| 员工类型描述（emp_cls_cd_desc） | 对应 `real_emp_cls_cd` 取值 |
| ----------------------- | --------------------------------------------------------------------------------------------------- |
| **正式** | `101` |
| **实习生** | `102`、`106` |
| **外包** | `105`、`115`、`116`、`122`（细分：**外包A** = `115`、**外包B** = `116`；其余两码为通称「外包」） |
| **其他** | `103`、`104`、`107`、`110`、`112`、`113`、`114`、`117`、`118`、`119`、`120`、`OTH`、**空**（NULL / 空串等未归入上述三类的取值） |

- **一类多码**：除「正式」外，筛选某一类时对 **`real_emp_cls_cd`** 使用 **INCLUDE** 与完整编码列表。  
- **「其他」中的空**：与离散码混合时，用 **`operator: "OR"`** 拆子条件，或对 NULL 单独使用 **`filterType: 8`（NU）** 等（见 `references/dipper/dipper-api.md`），**不要**把「空」误写为字面量 `"空"` 参与 IN（除非平台枚举确有该值）。

### 机器可读（Agent 展开用）

请求体中编码建议与线上一致为 **字符串**（如 `"101"`）。

```json
{
  "field_for_code": "real_emp_cls_cd",
  "categories": {
    "正式": { "codes": ["101"] },
    "实习生": { "codes": ["102", "106"] },
    "外包": { "codes": ["105", "115", "116", "122"] },
    "外包A": { "codes": ["115"] },
    "外包B": { "codes": ["116"] },
    "其他": {
      "codes": ["103", "104", "107", "110", "112", "113", "114", "117", "118", "119", "120", "OTH"],
      "includes_null_or_empty": true
    }
  },
  "defaults_by_metric_kind": {
    "hc": {
      "applies_to": "fieldType=1 且字段名含 hc/_hc_（如 ai_qa_est_hc、ai_qa_bgt_hc、ai_qa_est_rest_hc）",
      "trigger": "用户未显式指定员工类型时",
      "condition": { "field": "real_emp_cls_cd", "filterType": 10, "value": ["101", "115", "116"] },
      "user_facing_text": "员工类型范围：正式员工 + 外包A + 外包B"
    },
    "money_or_rate": {
      "applies_to": "金额、比率、使用率类指标（非 HC）",
      "trigger": "用户未显式指定员工类型时",
      "condition": "不加 real_emp_cls_cd 筛选（等价全集）",
      "user_facing_text": "员工类型范围：正式员工 + 外包 + 实习生 + 其他"
    }
  }
}
```

**condition 示例（筛「外包」）**：

```json
{
  "field": "real_emp_cls_cd",
  "filterType": 10,
  "value": ["105", "115", "116", "122"]
}
```

若用户原文与 **`emp_cls_cd_desc`** 一致且模型支持该字段筛选，可直接 EQ / INCLUDE；否则优先 **`real_emp_cls_cd` + 上表**。

## 城市区域（`loc_cty_desc`）

> **说明**：当前人力成本域（`bizId=119`）侧 **`loc_cty_desc` 仅下列取值**（与平台「城市区域」枚举一致时可直接用于筛选；**仅成本金额模型**支持该维度）。若线上下发增多或减少，以 **`data_search_dimension_values`** 为准并回写本表。筛名称时优先用本字段 **EQ** 或与 **INCLUDE** 组合（维度定义见 `measures_catalog.md` §一「分析维度」）。

### `loc_cty_desc` 合法取值一览

| 序号 | `loc_cty_desc` |
| --- | -------------- |
| 1 | 中国大陆其他 |
| 2 | 非中国大陆 |
| 3 | 上海 |
| 4 | 西安 |
| 5 | 南京 |
| 6 | 武汉 |
| 7 | 成都 |
| 8 | 重庆 |
| 9 | 北京 |
| 10 | 深圳 |

**condition 示例（单城市，如北京）**：

```json
{
  "field": "loc_cty_desc",
  "filterType": 1,
  "value": "北京"
}
```

**多城并查**（例如上海 + 深圳）可对同一字段使用 **`filterType: 10`**，`value` 为 `["上海", "深圳"]`（字符串须与上表**完全一致**）。
