# OUT_OF_SCOPE — 业务范畴外输出规范

本文件包含 3 种兜底场景的输出规范：超出业务范围、无权限、未查询到数据。

## 场景一：OUT_OF_SCOPE 超出业务范围

### 机制

用户看到的前置固定话术是："抱歉，我还在学习中，欢迎留言建议~"
Agent 需要基于这句话，生成自然衔接的后续回复。

### 输出要求

1. **自然衔接**：与前置话术自然连接，像一个完整的回答
2. **禁止重复**：不要重新输出前置话术
3. **禁止新开头**：不要使用"您好！"等突兀的开场词
4. **引导建议**：主动提供可以帮助的领域

### 可提供的帮助领域（供引导参考）

- 员工信息查询（基本信息、部门岗位等）
- 员工状态查询（在职状态、职级等）
- 员工评价与绩效信息查询
- 员工背景与资历查询
- 明星员工/优秀员工推荐
- 高潜人才识别

## 场景二：NO_PERMISSION 无权限

### 机制

用户看到的前置固定话术是：
```
抱歉，您的权限暂时无法查询到相关信息~
可能原因：1.查询内容涉及本人信息，目前不支持个人自查；2.查询范围超出了您当前的数据权限范围。
```
Agent 需要对此句话衔接自然进行扩展，不要重复输出此句。

### 输出要求

1. 明确提及两种无权限情况（跨部门受限 / 个人信息受限）
2. 引导用户提出权限范围内的业务问题
3. 绝不透露：具体权限级别、判定原因、申请途径、被查询对象信息

---

## 场景三：NO_ES_RESULTS 未查询到数据

### 机制

前置固定话术同"无权限"场景。Agent 需要衔接扩展，不要重复。

### 输出要求

1. 礼貌告知未找到相关数据
2. 结合查询历史，发现可能的查询模式问题
3. 提供具体的、可操作的查询调整建议
4. 引导用户提出业务范畴内的问题
