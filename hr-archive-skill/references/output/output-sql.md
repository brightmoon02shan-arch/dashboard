# 问数场景输出格式规范

> 所有 SQL 查询结果的最终输出必须遵循本规范。编写完 SQL 后必须通过 `sql_query` 工具实际执行，禁止只输出 SQL 不执行。

---

## 一、输出结构模板（四章节，严格按序）

每次问数输出由 **四个章节** 组成，按以下顺序排列：

```markdown
## {结合用户问题和权限信息生成的标题}

（权限说明：仅当标题因权限调整与用户原始提问不一致时展示。例如："您当前权限可查询范围为集团信息技术部，以下数据为该部门范围内的统计结果"）

{1-2 句关键发现摘要}

## 查询结果

| 维度 | 指标1 | 指标2 |
|------|-------|-------|
| XXX  | 123   | 45.6% |
| YYY  | 78    | 28.9% |

共计 XXX 人。

## 结论总结

{基于查询结果，围绕用户问题进行 2-4 句答案总结。突出关键发现、排名、趋势等}

## 统计口径

1. **数据统计时间**：本次数据为 YYYY 年 M 月 D 日快照数据
2. **回答依据说明**：本次统计结果严格依据您当前账号的数据查询权限生成
3. **数据权限范围**：查询权限仅限以下部门：{根据权限信息填充}
4. **核心指标说明**：{如涉及率类指标，用自然语言描述计算方式、公式}
```

---

## 二、各章节详细规范

### 2.1 标题章节

- 用 `##` 二级标题，内容为用户问题的**自然语言复述**，不是原始 query 的复制粘贴
- **标题必须结合用户 query 和权限信息综合生成**：当用户权限范围小于提问范围时，标题应直接体现实际可查范围
  - 用户问"集团离职率"但权限仅覆盖集团信息技术部 → 标题写"集团信息技术部离职率"
  - 用户问"各部门人数分布"但权限仅覆盖中国区 → 标题写"中国区各部门人数分布"
  - 用户权限覆盖提问范围 → 标题正常复述用户问题
- 如果标题因权限调整与用户原始提问不一致，在标题下方用括号说明权限约束
- 用 1-2 句话给出**关键发现摘要**，让用户不看表格也能快速获取核心结论

### 2.2 查询结果章节

查询结果以 **Markdown 表格** 为主要展示形式。根据查询类型采用不同表格结构：

#### 分布类（人数分布、占比分布）

```markdown
| 维度 | 人数 | 占比 |
|------|------|------|
| XXX  | 123  | 45.6% |
| YYY  | 78   | 28.9% |

共计 XXX 人。
```

#### 趋势类（月度/年度变化）

```markdown
| 月份    | 人数 | 环比变化 |
|---------|------|---------|
| 2026-01 | 120  | —       |
| 2026-02 | 135  | +12.5%  |
| 2026-03 | 128  | -5.2%   |
```

#### 排名类（各部门/序列排名）

```markdown
| 排名 | 部门 | 指标值 |
|------|------|--------|
| 1    | XXX  | 123    |
| 2    | YYY  | 98     |
```

#### 率类（离职率、流失率等）

```markdown
| 指标 | 数值 |
|------|------|
| 离职人次 | XXX 人 |
| 期末在职人数 | XXX 人 |
| 离职率 | XX.X% |
```

#### 名单类（人员明细）

```markdown
| 姓名 | 部门 | 岗位 | 职级 | 员工类型 |
|------|------|------|------|---------|
| 张三 | XX部 | XX岗 | 15   | 正式员工 |
```

#### 单值查询（总人数等简单统计）

- 不需要表格，直接用自然语言回答
- 例如："中国区当前在职员工共 12,345 人。"
- 单值查询无需"查询结果"和"结论总结"章节，直接在标题下方给出答案即可

### 2.3 结论总结章节

- 基于查询结果，**围绕用户问题**进行 2-4 句答案总结
- 问什么答什么，禁止添加用户未问到的维度分析
- 突出**关键发现**：最大/最小值、排名前列项、同比/环比显著变化
- 如涉及多个子问题，逐个给出结论

**润色规则**：

| 用户问法 | 总结方式 |
|---------|---------|
| "满足XX条件的有谁" | 先总结总人数，再列明细 |
| "XX的人数" | 直接给出总数 |
| "XX的分布" | 给出完整分布表格 + 重点维度点评 |
| "XX的离职率/流失率" | 给出率值，说明分子分母含义 |
| "同比/环比变化" | 给出两期数据 + 变化幅度 + 趋势判断 |

### 2.4 统计口径章节

> **⚠️ 强制要求：每一次回答都必须包含统计口径章节，无论查询简单或复杂，无论是否为追问，都不可省略。缺少统计口径属于输出格式错误。**

**每次查询必须附上统计口径说明**，包含以下要素：

| 要素 | 必填 | 说明 |
|------|------|------|
| 数据统计时间 | 是 | **必须给出具体日期**，如"2026 年 4 月 19 日快照数据"。从 SQL 结果中获取实际快照日期后填入，**严禁**输出"取 MAX(date)"、"最新快照日"等技术表达 |
| 回答依据说明 | 是 | 固定话术："本次统计结果严格依据您当前账号的数据查询权限生成" |
| 数据权限范围 | 是 | **必须列出具体部门名称**，从工具返回的权限信息（`user_permission`）中提取。如"查询权限仅限以下部门：中国区"。**严禁**写"当前查询已按您的数据权限自动过滤"等模糊表述 |
| 核心指标说明 | 是（涉及率类计算时） | 用自然语言描述，如"离职率 = 离职人次 ÷（离职人次 + 期末在职人数）× 100%" |

**追问场景的口径处理规则**：
- **属于追问**（如"那按部门呢"、"换成离职率看看"、"再细分到职级"等，查询对象和上下文连续）：统计口径**沿用上一次回答的口径**（时间范围、权限范围、统计范围保持一致），仅更新因查询变化而改变的部分（如新增的核心指标说明）
- **不属于追问**（主题完全不同、时间范围变化、查询对象切换等）：必须**独立判断并生成新的口径**，禁止沿用之前的口径

## 三、信息脱敏规则（红线）

### 3.1 严禁暴露的技术细节

| 类别 | 禁止出现的内容 | 正确替代 |
|------|---------------|---------|
| 表名 | `ads_ai_qa_emp_df`、`ads_ai_qa_trsf_tmn_f` | 不提及数据来源 |
| 字段名 | `emp_sts_cd`、`pos_lvl`、`hi_pfm_flg`、`tmn_eft_dt` | 用中文业务术语 |
| 员工类型编码 | `101`、`115`、`116`、`118` | 正式员工、外包A、外包B、外部顾问A |
| SQL 语句 | `SELECT ... FROM ... WHERE ...` | 不展示 SQL |
| 计算公式 | `tmn/(tmn+emp)` | 用自然语言描述 |
| 数据库术语 | 分区、快照、JOIN、CTE、GROUP BY、MAX(date) | 用业务语言 |
| 筛选条件代码 | `real_emp_cls_cd IN ('101','115','118')` | "统计范围：正式员工、外包A、外部顾问A" |
| 日期技术表达 | "取 MAX(date)"、"最新快照日" | 具体日期，如"2026 年 4 月 19 日" |

### 3.2 敏感数据脱敏

| 数据类型 | 脱敏规则 |
|---------|---------|
| 薪酬 | **绝对禁止**展示任何薪资绝对值 |
| 考勤 | 不展示具体工时/天数/排名位次，仅描述趋势 |
| 绩效等级 | **可用于查询筛选条件**，但输出中个人维度不展示 S/A/B/C/D 等级字母；分布统计场景可展示等级分布 |
| 代码产出 | 不展示 commit 数/代码行数，描述活跃度 |
| 人才评分 | 不展示具体分数，仅相对描述 |
| 内部评价 | 不逐字引用评审内容，仅综合概述 |

## 四、数值展示规则

### 4.1 数值一致性（最高优先级）

**答案中的所有数值必须与 SQL 实际运行结果完全一致。** 禁止任何形式的篡改、估算或编造。总结前必须逐项核对每个数值。

### 4.2 格式规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 整数 | 千分位分隔或中文单位 | "12,345 人" 或 "1.2 万人" |
| 百分比 | 保留 1 位小数 | "23.5%" |
| 平均值 | 保留 1 位小数 | "平均年龄 32.5 岁" |
| 率类指标 | 保留 1-2 位小数 | "离职率 4.8%"、"流失率 2.35%" |

### 4.3 大数字处理

- 万级以上使用中文单位：12,345 → "1.2 万人"
- 千级保留千分位：1,234 → "1,234 人"
- 百级以下直接展示：123 → "123 人"

### 4.4 空值处理

- 分组统计中空值/未知维度**必须**作为单独一行展示
- 使用业务化标签：如"无职级"、"未出绩效"、"未知学历"
- **禁止**静默丢弃空值记录

## 五、特殊场景处理

### 5.1 人员流动名单

展示离职/入职/转岗人员名单时：
- **必须包含员工类型列**，使用中文标签（正式员工/外包A/外部顾问A）
- 禁止只展示编码或省略此列

### 5.2 多问题查询

用户一次提问包含多个子问题时：
- 逐个回答，每个子问题的查询结果和结论分开展示
- 不可只回答部分问题就停止

### 5.3 数据截断

- SQL 返回超过 50 行时：展示关键数据 + 提示"结果较多，已展示前 N 条"
- 多维度交叉结果：优先展示最重要的维度，其余作为补充说明

### 5.4 空结果 / 报错

当查询返回空结果或执行报错时，使用标准话术：

```
抱歉，您的权限暂时无法查询到相关信息~
可能原因包括：
1. 查询条件过于严格，建议放宽筛选范围；
2. 查询范围超出了您当前的数据权限范围。
```

### 5.5 权限受限

当行级权限注入后实际可查范围小于用户提问范围时：
- 在标题章节下方说明实际可查范围
- 查询结果仅展示权限内数据
- 统计口径中说明数据权限范围

## 六、完整输出示例

### 示例 1：分布类查询

> 用户问：中国区各二级部门人数分布

```markdown
## 中国区各二级部门人数分布

中国区共有 XX 个二级部门，其中 XX 部人数最多，占比 XX.X%。

## 查询结果

| 二级部门 | 人数 | 占比 |
|---------|------|------|
| 互联网业务部 | 3,456 | 25.6% |
| 中国区销售部 | 2,345 | 17.4% |
| 手机部 | 1,890 | 14.0% |
| ... | ... | ... |

共计 13,456 人。

## 结论总结

中国区当前共有 13,456 名在职员工，分布在 XX 个二级部门中。互联网业务部人数最多，共 3,456 人，占比 25.6%；其次是中国区销售部 2,345 人（17.4%）和手机部 1,890 人（14.0%）。

## 统计口径

1. **数据统计时间**：本次数据为 2026 年 4 月 19 日快照数据
2. **回答依据说明**：本次统计结果严格依据您当前账号的数据查询权限生成
3. **数据权限范围**：查询权限仅限以下部门：中国区
4. **核心指标说明**：统计范围为正式员工、外包A、外包B
```

### 示例 2：率类查询

> 用户问：中国区2025年离职率是多少

```markdown
## 中国区 2025 年离职率

中国区 2025 年离职率为 X.X%。

## 查询结果

| 指标 | 数值 |
|------|------|
| 离职人次 | 1,234 人 |
| 期末在职人数 | 23,456 人 |
| 离职率 | 5.0% |

## 结论总结

中国区 2025 年全年共有 1,234 人离职，期末在职 23,456 人，离职率为 5.0%。

## 统计口径

1. **数据统计时间**：本次数据为 2025 年 12 月 31 日快照数据
2. **回答依据说明**：本次统计结果严格依据您当前账号的数据查询权限生成
3. **数据权限范围**：查询权限仅限以下部门：中国区
4. **核心指标说明**：离职率 = 离职人次 ÷（离职人次 + 期末在职人数）× 100%；统计范围为正式员工、外包A、外部顾问A；时间范围为 2025 年全年（2025-01-01 至 2025-12-31）；离职日期取生效日期
```
### 示例 3：同比类查询

> 用户问：中国区2025年高绩效流失率，同比2024年变化

```markdown
## 中国区高绩效流失率同比分析

|------|-------------|-------------|------------|

| 2025 年 | XX | XXX | X.XX% |

同比变化：X.XX 个百分点

## 结论总结

中国区 2025 年高绩效流失率为 X.XX%，较 2024 年（X.XX%）上升/下降了 X.XX 个百分点。高绩效员工流失情况整体呈XX趋势。

## 统计口径

1. **数据统计时间**：本次数据为 2026 年 4 月 19 日快照数据
2. **回答依据说明**：本次统计结果严格依据您当前账号的数据查询权限生成
3. **数据权限范围**：查询权限仅限以下部门：中国区
4. **核心指标说明**：高绩效流失率 = 高绩效离职人次 ÷ 高绩效在职人数 × 100%；统计范围为正式员工、外包A、外部顾问A；时间范围为 2024 年全年、2025 年全年；高绩效定义为绩效等级 A 及以上；同比变化为两年流失率之差（百分点）
```
