# 组织健康度诊断 — 工具使用指南

> **⚠️ 重要：组织健康度诊断使用 `org_health_report` 工具，禁止自行编写 SQL。**
> 工具内部执行确定性 SQL 查询，直接返回预渲染的 Markdown 报告。LLM 只需将 `[请根据...]` 占位符替换为基于数据的诊断和建议。

---

## 工具调用

```bash
mcporter call hr_talent_claw.org_health_report dept_name="部门名称"
```

- **dept_name**: 部门名称，支持任意层级（一级~六级），支持模糊匹配
- **sender_id**: 调用者 user_id（权限过滤）

### 工具返回值

成功时直接返回 **Markdown 格式的报告**，包含：
- 标题、牵引要求、数据截止信息
- 所有数据表格（层级分布、宽幅分布、偏窄/偏宽明细、管理者占比趋势、零下属明细）已预渲染
- 信号灯 emoji 已映射（🟢🟡🔴）
- `[请根据...]` 占位符标记 LLM 需要填写的诊断和建议部分

**LLM 的职责：将 `[请根据...]` 占位符替换为基于报告数据的专业诊断和建议，其余内容原样输出。**

部门匹配失败时返回 JSON：
```json
{"error": "'产品' 匹配到多个部门，请指定具体部门名称", "candidates": [{"name": "产品部", "level": 3}, ...]}
```

---

## LLM 输出规则

> **⚠️⚠️⚠️ 最高优先级规则：工具返回的 Markdown 报告必须原样输出！**
> 工具返回预渲染的 Markdown 报告（含标题、牵引要求、所有数据表格、分隔线、统计口径），LLM **原样输出报告全部内容**，仅将 `[请根据...]` 占位符替换为基于数据的专业诊断和建议。
>
> **绝对禁止以下行为：**
> - ❌ 将预渲染的 Markdown 表格转为纯文字描述
> - ❌ 重新组织报告章节结构（如自创 📊📐👔🎯💡 等新章节）
> - ❌ 删除牵引要求、统计口径等工具预渲染的内容
> - ❌ 用自己的格式重新排版数据
> - ❌ 输出中间推理过程
> - ✅ 唯一允许的修改：将 `[请根据...]` 占位符替换为诊断结论和建议

### LLM 需要填写的内容

工具返回的 Markdown 中包含以下占位符，LLM 必须替换为专业诊断：

1. **§1 诊断**（`💡 **诊断**：` 后的 `[请根据...]`）
   - 一级部门：层级深度结论 + 宽幅分布结论 + 偏窄/偏宽明细结论
   - 非一级部门：宽幅分布结论 + 偏窄/偏宽明细结论
2. **§2 诊断**：整体管理者占比结论 + YoY趋势结论 + 零下属人员结论
3. **§3 综合建议**：3 条具体建议，每条引用具体数字

### 输出规范

- **禁止中间过程输出**：收到工具数据后直接输出报告，禁止输出推导过程
- **禁止暴露技术细节**：表名、字段名、SQL、员工类型代码均不得出现
- **诊断语言面向管理者**：用"组织扁平"、"二传手风险"、"管理效能"、"管理层膨胀"等业务语言
- **建议要有数据支撑**：每条建议引用具体数字
- **报告其余部分原样输出**，不要修改工具预渲染的表格和数据
- **偏窄/偏宽明细、零下属明细**：每个明细表最多展示 60 人，按严重程度排序（偏窄：下属越少越靠前，偏宽：下属越多越靠前）。超过 60 人时工具会自动追加省略提示（如"共N人，仅展示前60人"），原样输出即可

## 层级人员明细 drill-down — `org_layer_detail` 工具

当用户在组织报告后追问某层级的人员明细或汇报链时使用。

### 工具调用

```bash
mcporter call hr_talent_claw.org_layer_detail dept_name="部门名称" layer=8
```

- **dept_name**: 部门名称，与 `org_health_report` 相同
- **layer**: 汇报层级编号，对应组织报告中的 L1-L8（如用户问"L8人员是谁"则传 `layer=8`）
- **sender_id**: 调用者 user_id（权限过滤）

### 触发场景

对话历史中最近一次调用了 `org_health_report`，且用户追问包含以下模式：
- "L8人员是谁"、"L5有哪些人"（L+数字+人员相关词）
- "第8层有哪些人"、"第6层的汇报链"（第N层+人员/汇报链）
- "哪些人在第6层"

### 工具返回值

成功时返回 Markdown 表格：

```markdown
**{部门名}L8 层级人员明细**（共 N 人）

| 姓名 | 所在部门 | 岗位 | 汇报链 |
|------|----------|------|--------|
| 张三 | XX部 | 高级工程师 | 雷军 → 卢伟冰 → ... → 张三 |
```

- 最多展示 60 人，按部门+姓名排序
- 汇报链为完整的汇报路径，`→` 分隔

**LLM 的职责：工具返回的 Markdown 表格直接输出即可，无需额外格式化。**

### 限制

- 仅支持一级部门（有层级分布数据的部门），非一级部门调用会返回错误提示
