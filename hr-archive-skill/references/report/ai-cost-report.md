# AI赋能人效分析报告（AI_COST_REPORT）

> **🚨🚨🚨 报告生成是两步流程，缺一不可：**
> 1. **调用 `ai_cost_report` 工具** → 获取报告骨架（sections JSON，含数据表格）
> 2. **填充分析 + 创建飞书文档** → 对每个 section 填充结论和分析，组装 Markdown，创建飞书文档
>
> **调用工具后必须立即执行第 2 步。** 禁止在调用工具拿到返回值后就停下、等待用户指令、或仅输出一段文字摘要。

---

## 一、工具调用

```bash
mcporter call --raw-strings hr_talent_claw.ai_cost_report dept_name="全公司" month="202604"
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `dept_name` | ✅ | 部门名称，支持集团/一级/二级…末级所有部门（如"全公司"、"手机部"、"集团信息技术部AI应用中心"） |
| `month` | 可选 | 报告月份，YYYYMM 格式（如"202604"），默认取上月（数据完整） |

### ⚠️ 自指性表述拒答

当用户使用"我部门"、"我的部门"、"我们部门"等自指性表述时，**必须直接拒答**，禁止调用工具。
回复话术："AI成本报告不支持'我部门'等自指性表述，请提供具体的部门名称，例如'全公司AI人效报告'、'手机部AI投入分析'。"

### 特殊场景处理

1. **用户未指定部门**：不调用工具，引导用户提供具体部门名称。
2. **"/"分隔的部门名称**：理解为层级关系，完整名称传给工具。
3. **部门找不到**：直接输出工具返回的拒答信息，不做重试，不自行替换。
4. **部门匹配多个候选**：工具返回 `{"error": ..., "candidates": [...]}` 时，必须将候选列表展示给用户选择。
5. **月份自动校正**：工具自动使用最新可用月份。LLM 无需做月份校验。
6. **集团级查询**：支持"全公司"、"小米集团"、"小米公司"。

## 三、Sections 填充规则

> **⚠️ 分析洞察原则**：禁止使用绝对阈值判断（如">5%为高"、"<3%为低"），必须从**对比**中发现洞察：
> - **同级部门对比**：排名、标杆 vs 洼地、分化程度
> - **趋势对比**：与上月环比、连续趋势、拐点判断
> - **结构变化**：渠道占比变动、部门排名变化

### Section 1: 整体结论

使用 `core_data` 中的数据，生成以下内容：

1. **核心数据摘要行**：
```
💰 人力月成本 {hr_cost_monthly_avg}万 | AI月成本 {ai_cost_monthly_avg}万 | AI/人力比 {ai_hr_ratio}% | AI渗透率 {overall_penetration}%
```

2. **4 个关键判断**（每个带 🟢/🔴/🟡 标识，基于对比而非绝对阈值，**不加序号、不加"判断1："等标题前缀**）：

**AI/人力成本系数**
- 与上月对比：上升/持平/下降
- 与同级部门对比：排名位置
- 🟢/🟡/🔴 标识基于趋势方向（上升🟡、持平🟢、下降🟢）

**AI成本增速**
- 环比趋势：连续上升🔴 / 趋缓🟢 / 波动🟡
- 与上月增速对比：加速/减速/持平
- 标识基于趋势变化（加速🔴、减速🟢、波动🟡）

**渠道结构**
- 各渠道占比与上月对比
- 结构性变化：某渠道占比变动>5pp 时重点提示
- 🟢 结构稳定 / 🟡 微调 / 🔴 结构性变化

**AI渗透率**
- 与同级部门对比：排名前3🟢 / 中位🟡 / 后3🔴
- 与上月对比：提升/持平/下降
- 标识基于排名位置和变化方向

3. **重点关注项**（从对比中发现的异常，2-3 条 + 1-2 条最有价值的管理动作建议）：
- 部门分化：渗透率最高 vs 最低差距>30pp 时预警
- 增速异常：环比增速变动>10pp 时预警
- 结构变化：某渠道占比变动>5pp 时提示
- 管理动作建议：只给出 1-2 条最有价值的，不逐条附管理动作

4. **统计口径声明**

### Section 2: Q1 双成本对比表

- 直接使用 `dept_comparison_markdown` 作为表格，**展示全部有数据的部门，禁止截取 Top10**
- **过滤规则**：人力成本和 AI 成本**均为 0**（或无数据）的部门不展示（两个条件同时满足才排除）
- 核心结论中列出 Top3 人力成本和 Top3 AI 成本部门

**分析解读（基于同级部门对比，4-5条）**：
1. 📌 **渗透率标杆 vs 洼地**：找出渗透率最高和最低的大部门（>500人），对比差异原因
2. 📌 **成本效率对比**：找出AI/人力比最高和最低的部门，分析投入产出差异
3. 📌 **人均成本分化**：找出人均AI成本最高和最低的部门，分析使用深度差异
4. 📌 **与上月对比**：找出渗透率变化最大的部门（上升/下降>5pp）
5. 💡 **行动建议**：基于对比结论，给出针对性推广策略

### Section 3: Q2 增长分析

- 使用 `monthly_trend_markdown` 和 `channel_breakdown_markdown` 作为表格
- 核心结论：月均增长率 + 渠道占比稳定性判断

**分析解读（基于趋势对比，2-3条）**：
1. 🔴 **增速趋势**：与上月环比对比 → 增速加速（环比上升）/ 趋缓（环比下降）/ 拐点出现（连续2月下降）
2. 📌 **渠道结构变化**：与上月对比，找出占比变动最大的渠道，分析变化原因
3. 💡 **行动建议**：基于增速趋势给出预算建议；基于渠道人均成本对比给出优化方向；基于渗透率部门对比找出推广重点部门

## 四、飞书文档创建

### 4.1 组装完整 Markdown

将 3 个 section 的内容按顺序组装为完整 Markdown 文档。**必须严格使用以下骨架，禁止自由发挥格式。** 所有 `{...}` 为占位符，替换为实际数据即可。

> **⚠️ 格式铁律**：
> - 表格使用标准 Markdown 表格 `| --- |`，禁止使用 `<lark-table>`（飞书卡片语法，DocxXML 不支持）
> - 整体结论的判断项和 Q1/Q2 分析解读**不加序号**，用 `-` 开头，emoji 放在文字前（如 `- 🟢 **内容**`）
> - 引用使用标准 Markdown `>` 语法，禁止使用 `<quote-container>`（飞书卡片语法，DocxXML 不支持）
> - 敏感数据警告使用 `<callout>` 高亮块
> - `⚠️ 重点关注项` 后**不加冒号**；该行前后必须各留一个空行、且行首无前导空格（否则飞书会把它当成上方列表项的延续内容，导致不换行）
> - 整体结论中**不逐条附管理动作**，只在重点关注项末尾给出 1-2 条最有价值的管理动作建议
> - 分析解读中的要点用 `1. 📌` 开头

> **⚠️ lark-cli 标题提取规则**：lark-cli 用 `@file` 传内容时，`--title` 参数会被忽略，标题从 markdown 正文的**第一行 `# 标题`** 提取。因此 markdown **必须以 `# 标题` 开头**，否则文档标题会变为 "Untitled"。

```markdown
# {dept_name}AI赋能人效分析报告（{year}年{month}月）

<callout emoji="🚨" background-color="light-red">
**人力成本属于公司最高级别敏感数据！请务必注意数据保密！请勿随意传播！导出的飞书文档仅操作人有阅读权限**
</callout>

> **部门范围：{dept_name}　　　　报告月份：{year}年{month}月**
```

## 五、红线规则

1. **禁止编造数据**：所有数字必须来自工具返回的 sections 数据，不得自行推算或编造。
2. **人数 < 5 不展示明细**：工具已处理，LLM 不需要额外处理。
3. **数据保密声明**：文档开头必须标注敏感数据警告。
4. **特殊一级部门不生成**：集团总办(PO)、集团总干部部(OD)、顾问群(CG)及其下属部门，工具会返回错误，LLM 直接输出拒绝信息。
5. **纯人力成本问题拒答**：如果用户问的是纯人力成本（如"XX部门成本报告"），路由到 COST_REPORT。
6. **多轮对话隔离**：每次报告请求独立处理，严禁引用对话历史中其他部门的数据。
7. **工具失败不兜底**：工具返回错误时，不得用 sql_query 替代生成报告。
8. **未来月份拒答**：工具会自动 clamp 到最新月份，LLM 无需额外处理。
9. **篡改数据禁止**：原样使用工具返回值，不得修改任何数字。
10. **口径区分（避免误导用户）**：
    - 人力成本金额为部门全口径实际成本（含所有员工类型、不限职级），与 COST_REPORT 成本报告口径一致，数字可对齐——若用户质疑两报告人力成本对不上，据此说明二者一致。
    - AI总成本直接从成本表计算，不 JOIN 员工表、不过滤在职/oprid。AI活跃人数（展示列）、人均AI成本通过 JOIN 员工表取部门归属，且**仅统计在职人员**（emp_sts_cd='A'）、过滤oprid为空，但**不**排除蓝领/外包A。**仅 AI渗透率**的分子（活跃人数）和分母（部门总人数）额外排除蓝领、中国区排除外包A。
    - 因此渗透率分子的活跃人数可能小于"AI活跃人数"展示列（前者在后者基础上额外排除蓝领/外包A）——这是口径差异，正常现象，无需调整。
    - 「人均AI成本」与成本报告「人均人力成本」分母口径不同，属不同指标，**禁止**引导用户将二者直接比较。
