# ES 通道查询拆解与执行编排

> **职责**：接收 `routing-es.md` 的意图识别结果，完成查询拆解（仅 BACKGROUND/TALENT）并编排下游检索与输出流程。
> **不含**意图识别逻辑——意图已由上游 `routing-es.md` 确定。

---

## 一、执行总览

```
意图识别结果（来自 routing-es.md）
  │
  ├─ OUT_OF_SCOPE ──→ 本文件不处理（已在路由层短路）
  │
  ├─ STAR_EVALUATION ──→ 跳过拆解 → 第二阶段（检索）→ 第三阶段（输出）
  ├─ STAR_STATUS     ──→ 跳过拆解 → 第二阶段（检索）→ 第三阶段（输出）
  │
  ├─ STAR_BACKGROUND ──→ 第一阶段（查询拆解）→ 第二阶段（检索）→ 第三阶段（输出）
  └─ STAR_TALENT     ──→ 第一阶段（查询拆解）→ 第二阶段（检索）→ 第三阶段（输出）
```

---

## 二、全局约束

- Thought 阶段仅描述中文语义，**禁止引用 ES 字段英文名与 DSL**
- Observation 阶段使用定性量词，**禁止具体阿拉伯数字**
- 迭代过程中不调用输出格式化，不对外输出；到达 Finish 判定时才调用
- 查询拆解结果仅作为下游 DSL 生成的语义输入，本阶段不产生任何 DSL

---

## 三、第一阶段：查询拆解（仅 STAR_BACKGROUND / STAR_TALENT）

在 Thought 中用中文语义完成拆解，**禁止出现任何 ES 字段英文名或 DSL 片段**。

### 拆解步骤

1. **用户意图概括**：一句话归纳任务目标

2. **三类项拆解**：

   | 类别 | 定义 | 处理规则 |
   |------|------|---------|
   | **必须满足项** | 人名、完整组织/部门、精确数值、明确状态 | 保持原样，禁止切分/同义替换/泛化 |
   | **可扩展项** | 宽泛能力、领域特征、偏好等语义条件（如"大模型经验"） | 仅按用户原始表达记录概念，同/近义词由下游 ReAct 迭代中补充 |
   | **待确认项** | 模糊评价（如"合适/理想"） | 不得直接臆断，需在后续步骤先查询相关背景/画像数据再推导 |

3. **其他限制/范围**（如年龄/工龄/司龄/职级/时间）：用中文语义表达区间或阈值

4. **策略提示**（可选）：粗判"关键字段 / 混合语义（RRF）/ 仅语义（KNN）"哪种更可能有效

### STAR_TALENT 专项规则

人才指示词（如"优秀员工/明星/顶尖/核心骨干/标杆/领军"等）在意图识别阶段已完成分类使命，查询拆解时应将其转化为**必须满足项**中的"人才类型"条件（注明用户是否区分应届/非应届），**不得**作为可扩展项的语义搜索概念传递给下游。查询中的其余非人才指示条件（如部门、背景技能等）按正常规则分类。

---

## 四、第二阶段：检索执行

**必须按顺序读取以下文件**（不可跳过、不可延后）：

| 步骤 | 文件 | 说明 |
|------|------|------|
| 1 | `es/es-retrieval.md` | 检索 Pipeline 主流程（查询分析 → DSL 构造执行 → 结果评估 → 质量门） |
| 2 | `es/field-insight.md` | **涉及部门/岗位/岗位序列时必须先执行**：聚合探查获取精确枚举值，再用于 DSL 构造 |
| 3 | `es/dsl-manual.md` | 字段 schema、强制规则、权重、完整 DSL 示例（**构造任何 DSL 前必须先读此文件**） |
| 4 | `es/dsl-star-background.md` | **仅 STAR_BACKGROUND 场景**：固定 DSL 模板，只替换关键词和 filter，禁止自行拼装 |

- 输入：意图判定结果 + 查询拆解结果（BACKGROUND/TALENT）或原始查询（EVALUATION/STATUS）
- Pipeline 内部完成：查询分析 → DSL 构造执行 → 结果评估 → 质量门 → 检索结束
- 输出：通过质量门的候选人列表

### 检索异常处理

- 待确认项非空时，首轮采用保守检索获取画像依据
- 无法合成有效 DSL 时，可回退调整查询拆解的覆盖面后重试
