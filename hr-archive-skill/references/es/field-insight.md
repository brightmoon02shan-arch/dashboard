# 枚举字段洞察工作流（Field Insight）

当查询涉及 `部门`（dept_name_path）、`岗位`（position_name）、`岗位序列`（position_seq）时，按以下流程获取精确检索值。

## ⚠️ 前置歧义检查（优先于所有步骤，红线规则）

**"A/B"歧义追问**：当用户使用 `/`、`和`、`、` 等分隔符提及多个部门时（如"人力资源部/汽车业务部里的业务楷模"），存在两种理解：① A部门下的B部门（层级限定）；② A部门和B部门（并列两个部门）。

**强制行为（违反视为严重错误）**：
1. **立即停止所有后续流程**，不得进入 Step 1-4，不得发起任何 `es_query` 调用（包括聚合探查、部门路径探查）
2. **直接向用户输出澄清问题**，格式如下，然后终止本轮输出：
   > 请确认您的查询范围：
   > 1. "{A}"和"{B}"是两个独立部门（分别查询）
   > 2. "{A}下属的{B}"是一个部门路径（层级限定）
3. **等待用户回复后**，根据用户选择重新开始执行完整查询流程

**禁止的绕过行为（历史上多次出现，逐条封堵）**：
- ❌ "先做聚合探查看实际数据，再决定是否追问" — 禁止，必须先追问
- ❌ "探查后发现某个部门不存在/无权限，歧义自动消除" — 禁止，权限范围不能替代用户意图确认
- ❌ "从语义/常识判断用户大概率是并列含义" — 禁止，不得替用户做决定
- ❌ 在 Thought/thinking 中识别出需要追问但选择跳过 — 禁止，识别出歧义就必须追问

## 涉及字段

| 字段名 | ES 字段 | 释义 |
|-------|---------|------|
| `position_name` | `position_name.keyword` | 岗位名称，仅用于职位检索 |
| `position_seq` | `position_seq.keyword` | 岗位序列，格式 `XX序列-XX类`，仅用于岗位序列检索 |
| `dept_name_path` | `dept_name_path.keyword` | 组织架构层级，格式 `一级部门-二级部门-XX-XX`，仅用于部门检索，路径包含即匹配 |

## Step 1：从用户 query 中提取关键词

分析用户对话历史，识别与上述枚举字段相关的关键词。例如：
- "企业效率部沈行怎么样" → 部门关键词: `企业效率`
- "研发序列的JAVA开发" → 岗位序列关键词: `研发`，岗位关键词: `JAVA`
- "中国区的零售市场部有哪些高潜应届生" → 部门关键词: 上级=`中国区`，目标=`零售市场`
- "国际业务部招聘组的人" → 部门关键词: 上级=`国际业务`，目标=`招聘`

**多级部门识别**：当用户表达包含"A的B部门"/"A下的B"/"A-B"等层级限定时，将部门关键词拆为**上级关键词**（用于缩小聚合范围）和**目标关键词**（用于 `include` 正则匹配）。无层级限定时只有目标关键词，走原有逻辑。

若用户未提及任何枚举字段相关条件，跳过后续步骤，所有字段输出空列表。

## Step 2：ES 聚合查询

对每个有关键词的字段，使用 `include` 正则在 ES 聚合时直接过滤候选值。

### 单级部门 / 岗位 / 岗位序列（默认模板）

```json
{
  "size": 0,
  "aggs": {
    "{field_name}_aggs": {
      "terms": {
        "field": "{field_name}.keyword",
        "size": 200,
        "include": ".*{关键词}.*"
      }
    }
  }
}
```

### 多级部门（存在上级关键词时）

当 Step 1 识别出上级关键词时，**必须**在聚合外层加 `query` 先缩小范围，再用目标关键词做 `include` 匹配：

```json
{
  "size": 0,
  "query": {
    "match_phrase": {
      "dept_name_path": "{上级关键词}"
    }
  },
  "aggs": {
    "dept_name_path_aggs": {
      "terms": {
        "field": "dept_name_path.keyword",
        "size": 200,
        "include": ".*{目标关键词}.*"
      }
    }
  }
}
```

> 无上级关键词时走默认模板，不加 `query`。
> **多字段可合并为一次调用**，在 `aggs` 中并列多个聚合即可。

## Step 3：从聚合结果中提取精确检索值

**通用规则**：
1. 对每个字段独立判断：字段释义与用户意图匹配 **且** 候选枚举值中存在可匹配项 → 需要检索，否则输出空列表
2. **仅从聚合返回的候选枚举值中挑选**，禁止修改或凭空编造
3. 每字段输出不超过 **10 个**检索值
4. 无匹配时输出空列表，不要强行匹配

**`dept_name_path` 特殊规则**：
- 末级路径优先匹配：优先选择枚举值**最后一级路径**与用户查询部门一致的枚举值
- 路径截断：若用户查询部门是枚举值路径的中间节点（子串），**截断到该节点层级**输出。多个匹配结果按匹配度排序，截断后去重
- **多路径歧义追问**：截断去重后，若同一个部门名称出现在**不同上级路径**下（如 `小米公司-人力资源部` 与 `小米公司-中国区-XX-人力资源部`），说明用户提到的部门存在歧义，**必须向用户追问具体指哪个部门**，列出候选路径供用户选择，禁止自行猜测。仅当所有候选路径的上级一致（只是下级展开不同）时，可直接截断输出，无需追问

**`position_name` / `position_seq` 规则**：
- 直接从候选枚举值中挑选，不做路径截断

## Step 4：输出

```
字段`position_name`推荐检索值：['JAVA开发', 'Java高级开发']
字段`dept_name_path`推荐检索值：['小米公司-企业效率部']
```

- 有输出的字段 → 在 ES DSL 中添加 `terms` 过滤（使用 `.keyword`）
- 空列表 / 未出现的字段 → 不加过滤条件

## 边缘情况

- ES 聚合无结果 → 跳过该字段
- 用户未提及任何枚举字段 → 所有字段输出空列表

## Few-Shot 示例

### 示例 1：岗位 + 部门组合查询

**用户 query**: "15级以上base北京曼彻斯特大学毕业的JAVA开发"

**字段提取结果**:
- `position_name` → `['JAVA开发', 'Java开发工程师', 'Java高级开发工程师']`
- `position_seq` → `[]`（用户未提及岗位序列）
- `dept_name_path` → `[]`（用户未提及部门）

### 示例 2：部门路径截断

**用户 query**: "人力资源部的王旭最近工作状态怎么样"

**字段提取结果**:
- `position_name` → `[]`
- `position_seq` → `[]`
- `dept_name_path` → `['小米公司-人力资源部']`

**说明**: 候选枚举值为 `['小米公司-人力资源部-新业务部', '小米公司-人力资源部-共享服务']`，按用户查询部门"人力资源部"截断到 `小米公司-人力资源部`。

### 示例 3：部门多匹配

**用户 query**: "国际业务部的王芳最近工作状态怎么样?"

**字段提取结果**:
- `position_name` → `[]`
- `position_seq` → `[]`
- `dept_name_path` → `['小米公司-人力资源部-国际业务部', '小米公司-国际业务部']`

**说明**: "国际业务部"在多条路径中出现，按匹配度输出多个截断结果。

### 示例 4：岗位序列查询

**用户 query**: "研发序列有哪些高潜人才"

**字段提取结果**:
- `position_name` → `[]`
- `position_seq` → `['研发序列-软件类', '研发序列-算法类', '研发序列-测试类']`
- `dept_name_path` → `[]`

### 示例 5：无枚举字段匹配

**用户 query**: "帮我看看张三的绩效"

**字段提取结果**:
- `position_name` → `[]`
- `position_seq` → `[]`
- `dept_name_path` → `[]`

**说明**: 用户只提到人名和绩效，不涉及任何枚举字段。
